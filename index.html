Agile

Agile is a mindset, it is more than just a collection of frameworks, XP. FDD, SCRUM, Sprints ect. Agile is an umbrella that covers all of these working methods, to be truly agile we must approach each journey with an open mind and allow the values and principles of agile development to guide our solution.

Test Driven Development

Never write a solution without a failed test.

The values that have defined TDD are as true now as they were when Kent Beck published eXtreme Programming (XP). First, you should write new business code only when an automated test has failed. Second, you should eliminate any duplication that you find. This ensures that the code I write is built of highly cohesive, loosely coupled components that are robust and easily maintainable.
